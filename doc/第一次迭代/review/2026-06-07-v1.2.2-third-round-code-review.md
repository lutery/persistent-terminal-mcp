v1.2.2 已把 `ptyTest()` 改成 `test.skip`，也移除了 5 个 `statusFile` 测试里的 Windows return，这是实质改进。但同一个文件里仍有 16 处普通 `test()` 或 `beforeEach()` 内部使用：

```ts
if (IS_WINDOWS) return;
```

这些测试在 Windows 上仍会被 Jest 计为 passed，而不是 skipped。开发报告写明“`No false-green tests (test.skip instead of return)`”，但当前代码不满足这个结论。

**影响**:

- `npx jest --runInBand` 显示 `26 skipped, 142 passed`，但其中仍包含没有执行主体逻辑的 passed 测试。
- 部分直接 return 的测试不依赖真实 PTY，例如 fake session 的写入行为和 non-existent terminal error path，本应可以在 Windows 执行。
- 测试报告仍不能完全反映真实覆盖状态。

**整改建议**:

1. 把仍需 Windows 跳过的测试统一改成 `ptyTest()` 或条件 `test.skip`。
2. 对 fake session / pure manager error path 测试移除 `if (IS_WINDOWS) return`，让它们在 Windows 上真实执行。
3. 重新跑 `npx jest --runInBand`，并确认没有 `if (IS_WINDOWS) return` 留在普通测试体中。
