## Remediation Fix Records

### Rx10: resume_terminal Command Injection [P0]

**Problem**: `resumeTerminal()` in `terminal-manager.ts:1126` directly interpolated `options.sessionId` into a shell command (`claude --resume ${options.sessionId}`) without sanitization. MCP, REST, and Web UI layers all passed sessionId through without format checks.

**Fix**:
- Added `validateSessionId()` function with whitelist regex `/^[A-Za-z0-9._:-]+$/` and max length 200
- Core validation at `terminal-manager.ts resumeTerminal()` entry point
- MCP layer: zod schema `.regex()` and `.max(200)` on sessionId
- REST layer: regex + length validation, HTTP 400 on failure
- Web UI layer: same validation, HTTP 400 on failure

**Files Modified**: `src/terminal-manager.ts`, `src/mcp-server.ts`, `src/rest-api.ts`, `src/web-ui-server.ts`

**Security Tests**: 13 test cases in `src/__tests__/resume-terminal-security.test.ts`:
- 5 valid (alphanumeric, hyphens, underscores, dots, colons)
- 8 invalid (ampersand injection, semicolon injection, pipe operator, spaces, newlines, quotes, max length exceeded, empty string)
- REST security test added to `src/__tests__/rest-api.test.ts`

**Verification**: All 13 security tests pass. Invalid sessionIds throw `INVALID_SESSION_ID` error. No terminal created on injection attempt.

### Rx12: Test False-Green Fix + statusFile Test Platform Skip [P1]

**Problem**: `ptyTest()` helper used `if (IS_WINDOWS) return;` which makes Jest count skipped tests as "passed" (false-green). 5 statusFile tests incorrectly skipped on Windows despite using fake sessions that don't need PTY.

**Fix**:
- `ptyTest()` changed from `return` pattern to `test.skip` pattern (`const maybeTest = IS_WINDOWS ? test.skip : test`)
- Applied to both `terminal-manager.test.ts` and `web-ui-server.test.ts`
- Removed `if (IS_WINDOWS) return;` from 5 statusFile tests (they use fake sessions, no PTY needed)

**Files Modified**: `src/__tests__/terminal-manager.test.ts`, `src/__tests__/web-ui-server.test.ts`

**Verification**: `npm test` now shows `Tests: 26 skipped, 142 passed, 168 total` — skipped count properly separated from passed.
