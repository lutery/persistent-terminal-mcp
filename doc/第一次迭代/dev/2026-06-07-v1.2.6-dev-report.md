## Remediation Fix Records

### Rx23A: Fix 3 silent returns in mcp-server.test.ts + rest-api.test.ts [P1]

**Problem**: `mcp-server.test.ts` (lines 21, 42) and `rest-api.test.ts` (line 70) used `if (process.platform === 'win32') return;` inside test bodies, causing Jest to count them as "passed" on Windows when they never executed their assertions.

**Fix**:
- Added `IS_WINDOWS` + `maybeTest` pattern to both files (same pattern as `terminal-manager.test.ts`):
  ```ts
  const IS_WINDOWS = process.platform === 'win32';
  const maybeTest = IS_WINDOWS ? test.skip : test;
  ```
- `mcp-server.test.ts`: Converted 2 PTY tests (`should return status for an active terminal`, `should return terminated status after killing terminal`) from `test()` to `maybeTest()`, removed silent return guards
- `rest-api.test.ts`: Converted 1 PTY test (`should create terminal without init options`) from `test()` to `maybeTest()`, removed silent return guard

**Files Modified**: `src/__tests__/mcp-server.test.ts`, `src/__tests__/rest-api.test.ts`

---

### Rx23B: Fix 6 REST PTY tests without Windows strategy [P1]

**Problem**: `rest-api.test.ts` lines 79, 100, 112, 142, 174, 200 create real terminals via `POST /terminals` but had no Windows skip strategy. These tests would either silently fail or produce `AttachConsole failed` noise on Windows.

**Fix**: Converted all 6 PTY-dependent tests to use `maybeTest()`:
- `should create terminal with init options and return init metadata` (line 79)
- `should return status for existing terminal` (line 100)
- `should accept includeOutputPreview query param` (line 112)
- `should wait for pattern and return result` (line 142)
- `should wait for task_result and attempt to parse` (line 174)
- `should accept adapter query param` (line 200)

Non-PTY tests (error paths, pure validation) remain as plain `test()`:
- GET / documentation, 404 for non-existent terminals, 400 for missing/invalid params, command injection test

**Files Modified**: `src/__tests__/rest-api.test.ts`
