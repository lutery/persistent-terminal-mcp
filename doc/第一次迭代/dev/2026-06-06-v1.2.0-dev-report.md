## 2. 新增功能清单

### 2.1 新增 MCP 工具 (4个)

| 工具 | 功能 |
|------|------|
| `wait_for_pattern` | 等待终端输出匹配正则表达式，支持捕获组、超时快照 |
| `wait_for_result` | 等待 `<task_result>` XML 块并解析为结构化结果(PASS/FAIL/ERROR) |
| `get_terminal_status` | 结构化状态快照，含语义状态检测(running/waiting_input/completed/error) |
| `resume_terminal` | 在新 PTY 中恢复 CLI agent 会话 |

### 2.2 增强现有 MCP 工具 (2个)

| 工具 | 增强内容 |
|------|----------|
| `create_terminal` | 新增 initCommands、readyPattern、readyTimeoutMs、initFailurePattern、statusFile 参数 |
| `read_terminal` | 新增 content_only 模式(过滤噪音)、last_response 模式(提取AI回复)、adapter 参数(generic/claude/codex) |

### 2.3 新增模块 (3个)

| 模块 | 文件 | 功能 |
|------|------|------|
| OutputFilter | src/output-filter.ts | 保守的TUI噪音移除，保留关键行(错误、文件路径、测试结果) |
| ResultParser | src/result-parser.ts | XML `<task_result>` 块检测与解析，禁用外部实体/DTD |
| StatusProvider | src/status-provider.ts | JSON 状态文件读取，zod 验证，非致命错误处理 |

### 2.4 REST API 扩展 (4个新端点)

- `GET /terminals/:id/status` — 结构化状态快照
- `POST /terminals/:id/wait-pattern` — 等待模式
- `POST /terminals/:id/wait-result` — 等待 XML 结果
- `POST /terminals/:id/resume` — 恢复终端会话

### 2.5 Web UI 扩展

- 终端列表页：语义状态徽章、相对活动时间
- 终端详情页：状态面板、输出过滤切换、wait-pattern/wait-result 操作、resume 操作
- 创建终端模态框：支持初始化选项
- WebSocket 事件：pattern_matched、status_changed

## 7. 已知限制

1. **Windows 信号限制**: `kill_terminal` 在 Windows 上不支持 SIGTERM/SIGKILL 等信号（node-pty 限制），相关终端管理器测试在 Windows 上有预期失败
2. **Claude/Codex 适配器依赖特定 UI 格式**: `last_response` 模式的 claude/codex 适配器依赖特定的 prompt 标记（`❯`/`●`），UI 变更可能导致适配器失效（会自动回退到 generic 模式）
3. **T15 真实 CLI 驱动自测**: 需要真实 Claude Code CLI 环境，属于手动/半自动测试，待用户环境就绪后执行
