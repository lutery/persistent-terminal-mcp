## 发现的问题

### 问题 1: wait_for_pattern 超时时间较长时返回大快照

**现象**: 120 秒超时后 snapshot 字段包含大量内容 (原始 5092 chars → 清理后 1620 chars)
**根因**: TUI 应用在 120 秒内产生大量输出（spinner 帧、重复渲染）
**影响评估**: P2 — 功能正确，但大快照可能影响上下文窗口效率
**建议**: 使用更精确的 pattern 或缩短 timeoutMs；content_only 模式已提供有效过滤

### 问题 2: resume_terminal init 时的 Escape 残留输入

**现象**: resume_terminal 创建新终端后，子代理收到残留的 "2" 输入（来自之前 Escape 键尝试）
**根因**: write_terminal 发送的 Escape 键在 TUI 中被解析为其他输入
**影响评估**: P3 — 不影响功能，子代理可识别并忽略无关输入
**建议**: 无需修复，属于交互操作的预期行为

### 问题 3: REST API 独立启动缺少 TerminalManager

**现象**: 独立启动 REST 服务器后，功能端点返回 "Cannot read properties of undefined" 错误
**根因**: REST 服务器需要 TerminalManager 实例，独立启动时未注入
**影响评估**: 非问题 — REST 服务器设计为与 MCP 服务器共享 manager 实例
**建议**: 无需修复
